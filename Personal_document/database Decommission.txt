Decommssion:--->
--------------------
pre requist:-

1. Open the change on BMC remedy -INF027-SQL Server Database Decommission
2. update decommission data in Records Manager (Update the decomissioned database in CA19/425859) -- Decommissioning and Migration Activities - 
Master List status as in Implemetation.
3. Determine if SQL Server Infrastructure is required to be decommissioned. If it is then this procedure should NOT be used.
4. Determine infrastructure components within the scope of the database decommissioning:
Application related databases.
SQL Server Logins associated with affected databases.
Jobs and Maintenance Plans related to the database.
Server Objects used by logic within the affected database.
Database Mirroring or Log Shipping Configuration.
DNS Alias with correct Server mapping.


Actions#
1.Connect the SQL Server Instance with the Database Host Alias Name. (Wherever applicable)
2.Disable specific database monitoring of the affected databases.
3.Arrange for application Tier to be stopped. Refer to the application specific notes on who does this.
4.Disable all application jobs in the SQL Server once they have completed running.
5.Disable the SQL Server jobs/Maintenance Plans related to application. Take the reference of application page for details.
6.Disable Transaction Log Shipping and Database Mirroring on the source database.
7.If Transaction Log Shipping is implemented on the affected database,remove Transaction Log Shipping.
8.If Database Mirroring is implemented on the affected database,remove database mirroring
	*Connect to the Database Engine of either mirroring partner.
 	*From the Standard bar, click New Query.
 	*Issue the following Transact-SQL statement:
12.Remove the Witness from a Database Mirroring Session 

Query:-
ALTER DATABASE <database_name> SET WITNESS OFF

Remove the Database Mirroring Session
ALTER DATABASE <database_name> SET PARTNER OFF

Alter the database with Recovery on Mirror Server
RESTORE DATABASE <database_name> WITH RECOVERY

13.Disable alerts associated with this database on both Principal and Mirror Server.
14. Once all connections to the database have been closed (ie. no active connections into the source database.)
then commence the decommissioning work. Otherwise confirm with application support that the application tier is down.
You can check whether connections exist into the database via activity monitor or running the below query in a query window.
If connections are persistent and unexplained then issue the kill command
OR 
or kill the session from within SQL Server Management Studio (SSMS).

14. Script::--->>>

 SELECT 
    DB_NAME(dbid) as DBName, hostname,
    COUNT(dbid) as NumberOfConnections,
    loginame as LoginName
FROM
    master..sysprocesses
WHERE 
    dbid > 0
GROUP BY 
    dbid, loginame,hostname
go


15. Disable the Application SQL Server Logins in the SQL Server. This is to ensure Middle tiers other than the application tier cannot access database and cause an error in the log. Wait for 10 - 15 minutes and review logs before recommencing decommissioning procedure.
16. Take the backup of Application jobs/Maintenance Plans, SQL Server Logins and server objects in the SQL Server infrastructure.
17. Change the Database into single_user_mode
18. Validate the size of the database and based on that initiate the backup on appropriate drive (E:\ or F:\).
19. In case of Standalone Server take the Native Backup of Databases on E:\Temp\CRQ_\<DatabaseName_yymmdd.bak>


BACKUP DATABASE [CMS]
TO  DISK = N'E:\Temp\CRQ_\<DatabaseName_yymmdd.bak'
WITH NOFORMAT, NOINIT,  NAME = N'<Database_Name>-Full Database Backup',
SKIP, NOREWIND, NOUNLOAD,  STATS = 1
GO


20.In case of Cluster take the Native Backup of Databases on F:\MSSQL10_50.<InstanceName>\MSSQL\backup\CRQ_\<DatabaseName_yymmdd.bak>
21.OFFLINE the database.

22.Delete alerts associated with this database on both Principal and Mirror Server.
23.Remove the database and associated application logins on the mirror/standby database server.
24.Drop Application Database jobs/Maintenance Plans, SQL Server Logins and server objects in the SQL Server infrastructure.
25. Detach the database.

USE [master]
GO
EXEC master.dbo.sp_detach_db @dbname = N'<Database_Name>'
GO

26. Take the backup of Application login passwords from the password portal.
27.Perform a long term backup of the Database jobs/Maintenance Plans, SQL Server Logins, server objects, Application login passwords, physical data and log files for the database and update the long term backups record in the BCC SQLSERVER LONG TERM BACKUPS wiki page.
